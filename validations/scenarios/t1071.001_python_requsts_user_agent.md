# T1071.001 - Python Requests User-Agent in Zeek HTTP

## Objective
Detect HTTP requests captured by Zeek where the user agent contains `python-requests`.

## Telemetry
- Zeek HTTP
- Host: SENSOR-NSM
- Data Source: Network traffic
- Platform: Security Onion

## Detection Logic
See `t1071.001_python_requests_user_agent.yml`

## Why It Matters
Adversaries may communicate using application layer protocols associated with web traffic to avoid detection/network filtering by blending in with existing traffic. Commands to the remote system, and often the results of those commands, will be embedded within the protocol traffic between the client and server.

## Expected Artifacts
- Zeek HTTP log entry
- `event.dataset: zeek.http`
- `user_agent` contains `python-requests`
- Source IP should map to the system generating the request

## Validation
1. On the attacker VM, run:
   `python3 -c "import requests; print(requests.get('http://10.10.10.20').status_code)"`
2. If no HTTP service is available on the target, start a temporary test web server on a reachable host.
3. In Security Onion Hunt, search for:
   `event.dataset: zeek.http AND user_agent: *python-requests*`
4. Confirm the request appears and verify source and destination details.

## Result
Pass

## Tuning Notes
This may also detect benign internal automation, administrative scripts, or developer tooling. Tune by source host, subnet, or expected destinations if needed.
