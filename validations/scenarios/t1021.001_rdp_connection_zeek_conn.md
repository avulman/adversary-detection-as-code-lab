# T1021.001 - Remote Desktop Protocol Connection in Zeek Conn

## Objective
Detect network connections to TCP port 3389 captured in Zeek connection logs.

## Telemetry
- Zeek Conn
- Host: SENSOR-NSM
- Data Source: Network traffic
- Platform: Security Onion

## Detection Logic
See `t1021.001_rdp_connection_zeek_conn.yml`

## Why It Matters
Adversaries may use Valid Accounts to log into a service that accepts remote connections, such as telnet, SSH, and VNC. The adversary may then perform actions as the logged-on user.

## Expected Artifacts
- Zeek connection log entry
- `event.dataset: zeek.conn`
- `destination.port: 3389`
- `network.transport: tcp`
- Source and destination IPs should reflect the initiating client and target host

## Validation
1. From a system that can reach a Windows host with RDP enabled, initiate an RDP connection to the target.
2. For example, connect to `WIN-ENDPOINT-01` on port 3389 using an RDP client.
3. In Security Onion Hunt, search for:
   `event.dataset: zeek.conn AND destination.port: 3389`
4. Confirm the connection appears and verify the source and destination details.

## Result
Pass

## Tuning Notes
This detection may produce benign hits from approved administration activity. Tune by source subnet, admin workstation list, or known management hosts if needed.
