# T1071.001 - Suspicious HTTP User-Agent Python Requests

## Objective
Detect HTTP requests using the python-requests user agent.

## Telemetry
- Suricata
- Protocol: HTTP
- Host: Sensor-NSM

## Detection Logic
See `t1071.001_http_user_agent_python_requests.rules`

## Why It Matters
Adversaries may communicate using application layer protocols associated with web traffic to avoid detection/network filtering by blending in with existing traffic. Commands to the remote system, and often the results of those commands, will be embedded within the protocol traffic between the client and server.

## Expected Artifacts
- HTTP traffic
- User-Agent contains `python-requests`
- Source IP = attacker VM
- Destination IP = internal or external web server

## Validation
1. On the attacker VM, run `python3 -c "import requests; print(requests.get('http://10.10.10.20').status_code)"` if Python requests is available
2. Alternatively, host a simple HTTP service and send a request with a `python-requests` user agent
3. Verify the alert appears in Security Onion Suricata alerts

## Result
Pass

## Tuning Notes
This may generate false positives if legitimate automation in the environment uses python-requests. Tune by source IP or destination if needed.
