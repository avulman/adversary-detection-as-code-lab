# T1021.004 - SSH Connection in Zeek Conn

## Objective
Detect SSH network connections to TCP port 22 captured in Zeek connection logs.

## Telemetry
- Zeek Conn
- Host: SENSOR-NSM
- Data Source: Network traffic
- Platform: Security Onion

## Detection Logic
See `t1021.004_ssh_connection_zeek_conn.yml`

## Why It Matters
Adversaries may use Valid Accounts to log into a service that accepts remote connections, such as telnet, SSH, and VNC. The adversary may then perform actions as the logged-on user.

## Expected Artifacts
- Zeek connection log entry
- `event.dataset: zeek.conn`
- `destination.port: 22`
- `network.transport: tcp`
- Source and destination IPs reflect the initiating client and target SSH server

## Validation
1. From a Linux host, initiate an SSH connection to a reachable system with SSH enabled.
2. Example:
   `ssh user@10.10.10.50`
3. In Security Onion Hunt, search for:
   `event.dataset: zeek.conn AND destination.port: 22`
4. Confirm the connection appears and verify source and destination details.

## Result
Pass

## Tuning Notes
This detection may produce benign hits from normal administration activity. Tune by expected admin hosts, management subnets, or approved automation systems.
