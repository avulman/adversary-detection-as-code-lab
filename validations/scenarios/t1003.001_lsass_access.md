# T1003.001 - LSASS Access

## Objective
Detect lsass.exe access

## Telemetry
- Sysmon Event ID 10
- Host: WIN-ENDPOINT-01
- Index: sysmon

## Detection Logic
See `t1003.001_lsass_access.spl`

## Why It Matters
Adversaries may attempt to dump credentials to obtain account login and credential material, normally in the form of a hash or a clear text password. Credentials can be obtained from OS caches, memory, or structures. Credentials can then be used to perform Lateral Movement and access restricted information.

## Expected Artifacts
- TargetImage = lsass.exe
- GrantedAccess = 0x1010, 0x1410 or 0x1F0FF

## Validation
1. Run `procdump.exe` to obtain lsass memory dump.
2. Run `mimikatz.exe` to analyze the lsass memory file.
3. Search Splunk with the detection query

## Result
Pass

## Tuning Notes
- 0x1010 - combination of PROCESS_QUERY_INFORMATION and PROCESS_VM_READ, classid "read LSASS memory". Commonly used by mimikatz, pypykatz, other lsass dump tools.
- 0x1410 - mainly same as above, seen in newer tooling
- 0x1F0FFF - full access to the lsass process, used by procdump and other aggressive dump tools
