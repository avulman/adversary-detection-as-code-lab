# T1003.001 - LSASS Access

## Objective
Detect lsass.exe access

## Telemetry
- Sysmon Event ID 10
- Host: WIN-ENDPOINT-01
- Index: sysmon

## Detection Logic
See `t1003.001_lsass_access.spl`

## Why It Matters
Adversaries may attempt to dump credentials to obtain account login and credential material, normally in the form of a hash or a clear text password. Credentials can be obtained from OS caches, memory, or structures. Credentials can then be used to perform Lateral Movement and access restricted information.

## Expected Artifacts
- TargetImage = lsass.exe
- GrantedAccess = 0x1010, 0x1410 or 0x1F0FF

## Validation
1. Run `Add-Type 'using System; using System.Runtime.InteropServices; public class X { [DllImport("kernel32.dll", SetLastError=true)] public static extern IntPtr OpenProcess(uint a, bool b, uint c); [DllImport("kernel32.dll")] public static extern bool CloseHandle(IntPtr h); }'; $lsassPid=(Get-Process lsass).Id; $h=[X]::OpenProcess(0x2000,$false,[uint32]$lsassPid); if($h -ne [IntPtr]::Zero){[X]::CloseHandle($h)}`
4. Search Splunk with the detection query

## Result
Pass

## Tuning Notes
Excluded splunkd.exe as it is the core executable process for Splunk software.
